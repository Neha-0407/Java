The system should support:

one-to-one messaging,
group conversations,
sending messages,
viewing conversation history,
unread message count,
notifications.

For a messaging system, low latency and availability are important.
However, we should still maintain reasonable consistency guarantees for message ordering and delivery within a 
conversation.

For this implementation, I’ll keep persistence in-memory unless we later want to discuss distributed storage or 
database integration.

However, notification dispatching is a good candidate for asynchronous processing using queues or event-driven architecture to reduce latency and improve scalability.
